KKR MailLens

Lokalny, szyfrowany indeks poczty z wyszukiwaniem SQLCipher + FTS5.

Opis

Aplikacja importuje pocztę ze źródła desktopowego obsługiwanego przez COM lub z serwera IMAP. Wiadomości trafiają do lokalnej, zaszyfrowanej bazy SQLite. Klucz aktywnej sesji istnieje wyłącznie w RAM procesu GUI.

Szybki start

  1. Uruchom run\KKR.MailLens.Gui.exe.
  2. Ustaw PIN i zainicjuj bazę.
  3. Odblokuj sesję.
  4. Uruchom import przyciskiem Harvest.
  5. Wyszukuj w GUI lub przez run\KKR.MailLens.exe query "neutralny tekst".

Neutralny przykład

  • temat: Test Record
  • nadawca: sender@example.invalid
  • odbiorca: recipient@example.invalid
  • treść: Neutralny tekst wiadomości używany do testowania indeksu
  • fraza: neutralny tekst

IMAP

Dane w przykładzie używają zarezerwowanej domeny .invalid i należy je zastąpić własną konfiguracją.

run\KKR.MailLens.exe imap-add --host imap.example.invalid --user sender@example.invalid
run\KKR.MailLens.exe imap-harvest --account sender@example.invalid --since 2026-01-01

Dane i bezpieczeństwo

Domyślny katalog to %LOCALAPPDATA%\kkr-maillens; zmienna KKR_MAILLENS_DIR pozwala wskazać inną lokalizację. SQLCipher, schemat SQLite, indeks FTS5 i idempotentny upsert pozostają częścią dotychczasowego przepływu.

Build

dotnet build KKR.MailLens.sln
dotnet run --project src\KKR.MailLens\KKR.MailLens.csproj -- selftest
